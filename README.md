# scooter_logic
Simple example of SOLID principles


Implementing imagined scooters availability system according to ![SOLID](https://dev.to/antonov_mike/how-can-applying-the-solid-principles-make-the-code-better-3fam).

Status:
- available
- reserved
- rented
- low battery
- out of service
- lost 

Client:
- customer
- employee 

Rental:
- regular
- discounted
- service
